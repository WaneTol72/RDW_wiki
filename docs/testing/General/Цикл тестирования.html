Цикл тестирования

Начало цикла

Визуальный осмотр устройства​

[!tip]+ Все изделия должны быть единообразны в рамках одного заказа/этапа.

  • Внешние дефекты устройства:​
    • для ПК. Сколы, царапины, вмятины, "мусор" в корпусе и т.п., Front панель - наличие всех необходимых заглушек/вставок, DvD привод (при наличии) и т.п., щели, соответствие номера штрих-кода (маленькая (корпусная) / большая(вложенная) наклейка) и их расположение на изделии.
    • для Моноблоков. Сколы, щели, царапины, "мусор" в корпусе и т.п., Front панель - заглушки/вставки, DvD привод (при наличии), единообразие сборки, экран матрицы на механические/физические повреждения, соответствие номера штрих-кода (маленькая (корпусная), большая(вложенная) наклейка) и их расположение, механизм веб-камеры, блок кнопок управления.
    • для MiniPC. Сколы, щели, царапины, "мусор/посторонний шум" в корпусе и .т.п., лицевая сторона / задняя сторона - ровное расположение звук/микрофон, USB, Type-C, и .т.п. соответствие и расположение наклеек (штрих-код).
  • Осмотр выходов Мат. плата/Motherboard/MB (для всех):​
    Язычки HDMI, хвостики USB, заглушка RJ45 Ровное расположение всех выходов (Type-C, звук, микрофон, и т.д.)
  • Подключение к тестовому стенду:​
    • для всех устройств: При включении обратить внимание на работу доп. кулеров и кулера ЦП, не должно быть посторонних шумов/свист/трещотки и т.п.
    • для ПК: Включение устройства при помощи кнопки Power Power (включение)- сразу проводится проверка подключения (исключение автостарт), Reset (перезагрузка)- можно проверить между каким-либо из этапов, при этом не увеличивая общее время тестирования/цикла тестирования устройства, Замок - при его наличии, проверить подключение (в положении: закрыт/открыт), Индикация - проверить при включении изделия и установки ОС (Power LED/HDD LED).
    • для Моноблоков: Power/Reset - проверка кнопок и индикации (Power LED/HDD LED) идентично ПК, Control Panel - проверить работоспособность блока кнопок управления (исключение: если меню в режиме разработчика, сделать сброс настроек). WebCam - проверить механизм открытия/закрытия (замок)
  • Регистрация изделия в системе учёта:​
    При помощи терминала, внести в систему учёта изделий введя свой ID сотрудника и штрих код изделия (на корпусе*)
  • Прошивка материнской платы (Motherboard/MB) опционально:​
    В зависимости от модели MB (AMD/intel) прошиваем BIOS через WDS/WinPE соответствующими командами, либо с флешки (актуальная версия/ТЗ).

BIOS​

  • Первоначальная настройка BIOS:​
    • Вход в BIOS AMD ==F2== intel ==del==
    • Boot menu AMD ==F12== intel ==F11==
    • для MiniPC вход в Bios ==del== вход в boot menu ==F7==
    • Важно!!!
      • для некоторых моделей MB (AMD/Intel) вход в BIOS и boot menu может отличаться Установка времени и даты* Проверка комплектующих и доп. настройки в соответствии с ТЗ, CPU, RAM, температура CPU, FAN1/FAN2/FAN3 RPM, HDD/SSD/M.2 Доп. настройки (в telegram).

[!WARNING] (Опционально) Включить пункт Boot from onboard LAN (UEFI: PXE)/Network stack/UEFI PXE Support.

Сохранить внесенные изменения BIOS: для всех ==F10== сбросить Optimized Default ==F9== для MiniPC ==F4== сбросить Optimized Default ==F3==​

  • Сброс остаточного питания (батарейка):​
    Выключить/отключить устройства (выдернуть кабель/шнур питания), прожать кнопку питания/сброс (Power/Reset) несколько раз для сброса остаточного питания. Подождать ~ 2 - 5 min, в зависимости от мощности БП (блок питания). Подключить и запустить устройство.
  • Запуск службы WDS/PXE:​
    Дождаться запуска PXE (в некоторых случая нужно через boot menu выбрать запуск службы UEFI: PXE IP4).

Установка Тестовой ОС​

  • Заливка тестовой операционной системы:​
    После того как загрузились в WDS/WinPE (окно выбора ОС). Открываем командную строку (cmd) ==Shift + F10== прописываем скрипт для очистки дисков (==000==), в этом же окне (cmd) прошиваем BIOS (при необходимости). закрываем всё лишние, заливаем тестовую ОС
  • Установка необходимых драйверов (опционально):​
    запускаем команду Выполнить ==Win + R==, свой ID сотрудника (для отображения в отчете устройства ID тестировщика), повторяем команду ==Win + R== (Выполнить), прописываем команды для работы с программой Veyon Master (опционально), установка необходимых драйверов производиться программой SDI (опционально)

[!WARNING]+ Исключение: некоторым устройствам нужно скачать драйвер отдельно с официального сайта

Разъемы и доп. устройства​

  • Цикл проверки разъёмов и дополнительных устройств:​
    Front Panel - Проверка разъемов USB, Type-C, Card reader, звук/микрофон и т.п. DvD - Проверить на открытие/закрытие механически (кнопка), а так же программно, открыть Мой компьютер(правый клик мыши)/< Извлечь> MB - проверка разъемов Flex - при наличии, USB, Type-C, звук/микро, доп. платы расширений, и т.п. (может быть опциональным*) на моделях MB Intel для разъема Type-C проверятся вывод изображения на отдельный/внешний монитор (это может быть опциональным*) Wi-Fi/BT(Bluetooth)* - проверить работоспособность/сигнал (для PC, при его наличии на изделии) [[Плата защиты|ПЗПК]] - проверка плата защиты (при его наличии) [[Видеокарта]] - все выходы проверяются без исключения.
    • для Моноблоков: Матрица - проверка на пиксели, артефакты, засветы, и т.п. DvD - Проверить на открытие/закрытие механически (кнопка), а так же программно, открыть Мой компьютер(правый клик мыши)/< Извлечь> Внешний звук - динамики/колонки (правильно положение/вывод звука Left/Right) WebCam - выставить актуальные настройки в соответствии с ТЗ , Wi-Fi/BT(Bluetooth) - проверить работоспособность/сигнал.
    • для MiniPC: процедура проверки идентична проверке PC ,+ проверка Wi-Fi/BT при условии, что данное оборудование установлено на изделии.

Стресс тесты​

  • AIDA - стресс тест проводится ~15 min на 10% от общего объема партии заказа, на оставшейся партии ~5 min, параллельно запускаем информатор HWInfo .
  • FurMark - для видеокарт проводится так же ~15 min на 10% от общей партии заказа, на оставшейся партия заказа ~5 min

Снятие отчета​

Перед снятием отчета, уточнить у коллег, не снимает ли кто еще параллельно отчет (чтобы после снятия отчета, они не перемешались). Снятие отчет производится через вызов команды ==Win+R== в поле вводим свой ID сотрудника, либо через ярлык, в общей папке на рабочем столе (выбираем свой). Перед снятием отчета, нужно произвести актуальные установки (наименование заказа (1), тип устройства (2), номер заказа (3), выбор модели МВ (5).) . После снятия, проверяем файлы отчета на идентичность.

Удаление тестовой ОС​

Перезагружаем устройство, заходим в WDS/PXE, дождаться окна выбора установки ОС (на всех устройствах), нажимаем комбинацию клавиш
==Shift+F10== (вызов командной строки/cmd), в окне вводим "000" данная команда очищает локальные диски (диск 0, диск 1, диск 2) от тестовой ОС.

[!tip]+ Рекомендация: Перед вводом этого скрипта ("000"), вытащить все флешки из изделия.
